### 私募基金收益计算模型
- 批次数据结构：`{ id, date, amount(万), costNav, note, exitDate?, exitNav? }`
- 浮动盈亏 = (当前净值 - 成本净值) × 份额（仅未退出批次计入）
- 已实现盈亏 = 分红收益 + 退出时资本利得（退出时结算）
- 业绩报酬 = max(0, 退出时资本利得) × 业绩报酬比例（仅退出时计提）
- 总净盈亏 = 浮动盈亏 + 已实现盈亏 - 业绩报酬

### 2026-05-09 年化波动率计算修复（第三次）

**问题**：年化波动率显示异常——先是在 196%~451%（严重虚高），后降到 2.6%~4.6%（严重偏低），还有 NaN

**根因分析**：
1. 原始"日历天数法"公式错误
2. 第一次修正为 `std(ln(1+r)) × √252`，没考虑数据频率差异（周频数据虚高2倍）
3. 第二次修正为时间加权法，但除以 `days` 后周频数据被低估（√7 ≈ 2.65倍）
4. `Math.log(1 + r)` 在 `r <= -1` 时返回 NaN

**最终方案：对数收益率 × √(365/平均间隔天数)**：
```js
avgDays = Σ(days) / N
logReturns = segments.map(s => r <= -1 ? 0 : Math.log(1 + s.r))
variance = std(logReturns)² // 样本方差
periodsPerYear = 365 / avgDays // 日频=365, 周频=52, 月频=12
annualizedVol = √(variance × periodsPerYear)
```

**测试验证**（模拟数据年化波动率 ~20%）：
| 数据频率 | 结果 | 正确？ |
|----------|------|--------|
| 日频（1天间隔） | 18.28% | ✅ |
| 周频（7天间隔） | 18.78% | ✅ |
| 月频（30天间隔） | 17.70% | ✅ |
| 混合频率（2.8天平均） | 18.85% | ✅ |

**重要教训**：
- 函数名 `calcPerformance`（不是 `computePerformance`），只有1个参数
- `Math.log(1 + r)` 必须防御 `r <= -1` 的情况
- 年化波动率必须根据数据实际频率选择年化因子，不能用固定的 √252

### 2026-05-09 年化波动率 NaN 修复（第四次）

**问题**：归因分析"净值拟合"表格中，4个基金（中柏夏至/深州正行7号/倍数多资产2号/日出15号）显示 `NaN%`

**根因**（两层）：
1. **计算层**：`cumNav` 为 0/undefined 时除零产生 NaN/Infinity，`Math.log` 收到 NaN 后传播到整个波动率
2. **显示层**：`fmtVal` 只检查 `v == null`，不检查 `isNaN(v)`；且 `null * 100 = 0` 导致波动率为 null 时显示 `0.0%` 而非 `--`

**修复（4层防护）**：
1. segments 构建时跳过 `!prevCum || !currCum || !isFinite(...)` 的无效段
2. logReturns 中增加 `!isFinite(r)` 防护
3. 最终结果 `safeVol = isFinite(vol) && vol >= 0 ? vol : null`
4. `fmtVal` 增加 `isNaN(v)` 检查；波动率单元格先判 `!= null` 再做乘法
