## 净值拟合 95% VaR 修复

**问题**：拟合统计表格中 95% VaR 全部显示为正数（+1.3%、+2.7% 等），金融意义上错误。VaR 应表示"最大可能损失"，应为负数。

**根因**：
1. **计算层**：`var95 = 1 - Math.exp(sortedLog[idx95])`，当对数收益率为负时，此公式反而得到正数（如 `1 - 0.987 = +0.013`）。
2. **显示层**：`fmtPerf` 中硬编码了 `'+'` 号，且颜色固定为红色。

**修复**：
- 第5894行：`1 - Math.exp(sortedLog[idx95])` → `Math.exp(sortedLog[idx95]) - 1`
- 第3427行：去掉硬编码 `+` 号，颜色逻辑改为 `${p.var95 >= 0 ? 'var(--red)' : 'var(--green)'}`

修复后 VaR 正确显示为负数（如 -1.3%），绿色标注，符合"风险/损失"含义。

## 拟合统计表格颜色统一修正

**问题**：拟合统计表格各列颜色规则混乱——收益率列按A股惯例（正红负绿），但比率列却是反的（正绿负红），且最大回撤硬编码红色、信息比率和盈亏比无颜色。

**修复**（`fmtPerf` 函数，统一为A股惯例：正=红，负=绿）：
1. **最大回撤**：`var(--red)` 硬编码 → 按数值正负变色
2. **夏普比率**：正绿负红 → 正红负绿
3. **卡玛比率**：正绿负红 → 正红负绿
4. **信息比率**：无颜色 → 正红负绿
5. **索提诺比率**：正绿负红 → 正红负绿
6. **盈亏比**：无颜色 → ≥1 红色，<1 绿色
7. **正收益占比**：保持中性无颜色（合理）

## 横向对比口径统一（统计 + 曲线一致）

**问题**：用户发现勾了横向对比后，曲线从共同起点归一化，但拟合统计表格数据不变（仍基于全部数据），口径不一致。

**修复**：

1. **提前计算 `commonStartDate`**：在 `selFundStats` 之前（`selFunds` 之后），遍历所有选中基金，取最晚的第一个日期作为共同起点（仅当勾了横向对比且没选区间起点时）
2. **统计表格也基于共同起点**：`effectiveStartDate = navFitStartDate || commonStartDate`，传给 `calcPerformance`
3. **`normalizedSeries` 复用**：移除重复的 `commonStartDate` 计算，直接使用提前计算的值

**结果**：
- 勾了横向对比 → 曲线和统计都从共同起点算
- 不勾横向对比 → 各自从最早日期算
- 选了区间起点 → 忽略横向对比，从区间起点算（已有行为）
