## 净值拟合页面修复：清空按钮 + 统计表格按选择过滤

**问题**：
1. 点击「清空」按钮后基金仍然全选 — `renderNavFit` 中 `size === 0` 时自动全选所有基金
2. 拟合统计表格始终显示全部基金，与上方选择无关

**修复**：
1. 移除 `selectedNavFitFunds.size === 0` 时的自动全选逻辑，清空后 truly 清空
2. 统计表格和汇总卡片改为只基于 `selFunds`（选中的基金）计算和显示
3. 未选基金时表格显示提示："请在上方选择至少1只基金以查看拟合统计"

## 净值拟合页面修复：选中基金后净值曲线不显示

**问题**：选中基金后，汇总卡片和统计表格有数据，但净值曲线区域始终显示"请选择至少1只基金以显示净值曲线"

**根因**：`selFunds.length === 0` 时，代码执行 `parentElement.innerHTML = '...'` 物理删除了 `<canvas id="navfit-chart">` 元素。后续即使选中基金，`document.getElementById('navfit-chart')` 返回 null，Chart.js 报错导致函数中断，图表区域保留上次的空状态消息。

**修复**：
1. 两处空状态处理（初始无数据 / 选中为空）改为隐藏 canvas + 动态创建空消息 div，不再删除 canvas
2. 图表渲染前增加保护：显示 canvas、移除空消息 div
3. 同时修复 `selectedNavFitFunds` 中 ID 类型不一致问题（dataset 始终是字符串，统一用 `String()` 转换）

## 净值拟合页面：添加日期范围选择器

**需求**：查看基金在特定时间范围内的表现

**实现**：
1. 在控件区添加开始日期/结束日期两个 `<input type="date">` 选择器 + 重置按钮
2. 新增状态变量 `navFitStartDate` / `navFitEndDate`
3. 修改 `calcPerformance(fund, startDate, endDate)` 支持日期范围过滤
4. `renderNavFit` 中：
   - 统计表格调用 `calcPerformance(f, navFitStartDate, navFitEndDate)`
   - 图表数据构建时过滤范围内的净值点
   - 图表描述和表格脚注显示当前日期区间
5. 日期变更自动触发重新渲染

## 净值拟合：同一天净值点不在同一根垂直线上

**问题**：截图显示两只基金同一天的 tooltip，但两个圆点不在同一根垂直线上

**修复**：将数据集里的 `x: p.date`（字符串）改为 `x: new Date(p.date + 'T00:00:00Z').getTime()`（时间戳），让 Chart.js 用统一时间戳对齐。

## 净值拟合：tooltip 有时只显示一条曲线

**问题**：两只基金更新日期不完全一致，tooltip `mode: 'index'` 只在精确同一天有数据时才显示两只

**修复**：改成 `mode: 'nearest', axis: 'x'`，同一鼠标位置每只基金都显示 X 轴上最近的数据点。

## 净值拟合：添加5个高级绩效指标

**新增指标**（在 `calcPerformance` 和表格中）：
1. **索提诺比率**（Sortino Ratio）— 只用下行波动率，对私募更合理
2. **盈亏比** — 平均正收益段 / 平均负收益段（绝对值）
3. **最大回撤修复时间** — 从回撤底部回到前期峰值所需天数
4. **最长连盈/连亏天数** — 按日历天数计算
5. **95% VaR** — 在险价值，95%置信度下的最大损失比例

**涉及修改**：
- `calcPerformance()`：新增5个指标计算逻辑
- `fmtPerf()`：新增5列 `<td>` 输出
- 拟合统计表格：新增5个 `<th>` 表头
