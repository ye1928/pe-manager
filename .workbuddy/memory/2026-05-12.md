## 组合模拟区间收益率修复（2026-05-12）

**问题**：3只等权重基金（+50.9%、+10.6%、-1.5%），组合区间收益率显示 +252.6%（应为约 +20%）

**根因**：
- 原始代码直接用基金原始净值加权，不同基金净值基数差异巨大
- 数据起始日期不一致时，早期组合净值只包含部分基金，baseNav 异常偏小
- `finalNav / baseNav` 比值被严重放大

**修复**：
1. 取所有基金都有数据的共同日期区间（`commonDates`）
2. 每只基金在共同区间内归一化到 1.0 起点
3. 组合净值 = Σ(归一化净值 × 权重)
4. stats table 和 chart 中单只基金也用共同区间计算
5. 共同日期不足2天时显示提示信息

## 组合模拟 commonDates 逻辑优化 + 日期范围显示（2026-05-12）

**问题**：
1. 加了7只基金但图表显示空白 — `commonDates` 要求所有基金每一天都有数据，某只基金缺一天就全部无法计算
2. 没有显示日期范围 — 用户不知道区间从哪到哪

**修复**：
1. 改为取各基金数据区间的**交集**（最晚起始日 → 最早结束日）
2. 区间内缺失净值用前值填充（前向填充），不再要求每一天都有数据
3. 汇总卡片下方添加日期范围提示："计算区间：2024-01-01 至 2025-01-01（共 365 天）"
