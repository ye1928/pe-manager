## Bug 修复：业绩报酬显示逻辑

**问题**：合计已实现盈亏弹窗中，顶部显示业绩报酬 -3.14万，但基金明细行都显示"--"，逻辑对不上。

**根因**：`totalPerfFee` 包含了两类业绩报酬：
- 已退出批次：从退出时的资本利得计提
- 持仓批次：从浮动盈亏计提

原代码在所有视图都用 `totalPerfFee`，但基金明细行只展示 `realizedGain`（持仓基金为0），导致合计对不上。

**修复方案**（已完成）：
1. `calcFund` 添加 `totalRealizedPerfFee` 字段（仅已退出批次计提）
2. `getPEStats` 添加 `totalRealizedPerfFee` 汇总
3. `openPnlDetail`：
   - 浮动视图：业绩报酬列显示 "-"（浮动无业报）
   - 已实现视图：业绩报酬只用 `totalRealizedPerfFee`
   - 净视图：业绩报酬用 `totalPerfFee`
4. `showRow` 逻辑：有 `realizedGain > 0` 或 `totalRealizedPerfFee > 0` 时才显示基金行
